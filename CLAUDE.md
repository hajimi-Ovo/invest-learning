## Obsidian 双链 & 块引用规则

### 块引用 ID 规范（重要）

- ❌ **禁止使用中文**作为块引用 ID（如 `^内在价值`）— Obsidian 对中文块 ID 的预览支持不可靠
- ✅ **优先使用 Obsidian 自动生成的** `^xxxxx` 格式 ID（在目标段落行尾输入 `^` 即可自动生成）
- ✅ 表格**单元格内不能放块引用** — 块引用必须放在独立行或行尾，不能在表格单元格内

### 双链锚点优先级

1. **块引用 `#^xxxxx`** → 最优，可精确预览段落内容
2. **标题锚点 `#标题名`** → 次优，可预览章节内容
3. **纯笔记链接 `[[笔记名]]`** → 基础，预览从笔记开头显示

## 数学符号格式约定

### LaTeX 公式写法

- **显示公式**（独立成块）：使用 `$$ ... $$`（而非 `\[ ... \]`）
- **行内变量**（嵌入中文段落）：使用 `$...$`（而非 `\(...\)`）
- 所有数学变量、字母符号必须用 `$...$` 包裹，确保在笔记中正确渲染

### 示例

```markdown
<!-- ✅ 正确 -->
均价线 $\text{AMP}$ 的计算公式为：
$$
\text{AMP}(t) = \frac{\sum_{i=1}^{n} P_i \times V_i}{\sum_{i=1}^{n} V_i}
$$
当 $\Delta > 0$ 时，$P_{\text{now}}$ 向 $\text{AMP}$ 回归。

<!-- ❌ 错误 -->
均价线 \(\text{AMP}\) 的计算公式为：
\[
\text{AMP}(t) = ...
\]
```

## 笔记维护惯例

- 每日笔记放在 `July/` 或 `June/` 月度文件夹下
- 添加 YAML frontmatter：`date`, `tags`, `type`, `status`
- 关联笔记区使用 `[!info]` 或 `[!tip]` callout 汇总

## 隐私安全与 Git 推送规范

### 禁止上传的内容

| 类别 | 示例 | 处理方式 |
|------|------|---------|
| **个人信息** | 手机号、邮箱、家庭地址、身份证号 | 绝对禁止上传，发现立即清除历史 |
| **账户凭证** | 密码、API Key、Token、私钥、OTP | 绝对禁止上传，发现立即轮换凭证 |
| **本地配置文件** | `.env`、`*.local.*`、`settings.local.json` | 加入 `.gitignore`，禁止提交 |
| **会话数据** | `data.json`（插件运行缓存） | 已在 `.gitignore` 排除 `plugins/*/data.json` |
| **个人工作区布局** | `workspace.json`、`graph.json` | 已在 `.gitignore` 排除 |

### 推送前检查清单

1. 运行 `git status` 确认没有意外包含配置文件
2. 检查新增/修改的 `.md` 文件是否包含个人身份信息
3. 图片文件（`attachment/*.png`）不包含屏幕截图中的个人信息
4. `.obsidian/` 目录下的文件原则上不应提交，除非确认为公共配置
5. 首次推送或批量添加文件时，先 `git add --dry-run` 预览待提交内容

### 隐私泄露应急处理

若发现隐私文件已上传至 GitHub：
1. **立即轮换凭证**（如有密码/API Key）
2. 使用 `git filter-branch` 或 BFG Repo-Cleaner 从 Git 历史中彻底清除
3. 强制推送清理后的历史：`git push --force`
4. 通知 GitHub Support 删除缓存的提交数据（如有必要）

## 笔记整理工作流（Claude 执行标准）

每次处理新笔记时，严格按以下流程执行：

### 流程步骤

1. **内容审校** — 检查原始内容的正确性（事实错误、逻辑矛盾、表述不清），相当于编辑角色
2. **格式检查** — 确认 Markdown 层级结构、LaTeX 公式规范、Callout/表格等元素正确
3. **添加 Frontmatter** — YAML 属性块，包含 `date`、`tags`、`type`、`status` 等字段，参考 vault 中既有笔记格式
4. **添加深度双链** — 核心环节，见下方详细规则
5. **推送至 GitHub** — 整理完成后提交并推送

### 深度双链规则（核心要求）

- ❌ **杜绝**单纯的整篇笔记链接 — 跳转后还需手动寻找对应知识点位置，浪费时间
- ✅ **必须精确到具体语块**，按优先级：
  1. **块引用 `#^xxxxx`**（优先使用 Obsidian 自动生成的 ID）→ 最优，可精确预览段落
  2. **标题锚点 `#标题名`** → 次优，可预览章节内容
  3. **纯笔记链接 `[[笔记名]]`** → 仅当该笔记全篇围绕同一主题时使用
- 🎯 追求知识点的 **深层内在关联性**，而非浅层的主题归类
- ⚠️ 特别留意**弯引号问题**：链接锚点中的引号必须与目标标题使用的引号类型一致（通常用直引号 `"` 0x22），避免 Obsidian 因字符不匹配找不到锚点
