
下面是**最基础、最常用**的 Markdown 语法，学会这些就能覆盖 90% 的使用场景。

---

### 1. 标题
使用 `#` 符号，有几个 `#` 就代表是几级标题，一共支持 6 级。

```markdown
# 一级标题
## 二级标题
### 三级标题
#### 四级标题
##### 五级标题
###### 六级标题
```

---

### 2. 段落与换行
- **段落**：直接连续书写文字，段落之间用**空行**隔开。
- **换行**：在行末尾加上**两个或以上空格**，然后回车；或者直接使用 `<br>` 标签。

```markdown
这是第一段。
（这里有一个空行）

这是第二段。

这是第一行末尾有两个空格
这是换行后的内容。
```

---

### 3. 强调（字体样式）

| 效果         | 语法                  | 示例        |
| :--------- | :------------------ | :-------- |
| **加粗**     | `**文字**` 或 `__文字__` | **粗体**    |
| *斜体*       | `*文字*` 或 `_文字_`     | *斜体*      |
| ***加粗斜体*** | `***文字***`          | ***粗斜体*** |
| ~~删除线~~    | `~~文字~~`            | ~~删除线~~   |

---

### 4. 列表

#### 无序列表
使用 `-`、`+` 或 `*` 作为标记（后面要跟一个空格）。

```markdown
- 苹果
- 香蕉
  - 香蕉的子项目（缩进 2 或 4 个空格）
- 橘子
```

显示效果：
- 苹果
- 香蕉
  - 香蕉的子项目
- 橘子

#### 有序列表
使用数字加英文句点 `.`。

```markdown
1. 第一步
2. 第二步
3. 第三步
```

---

### 5. 链接

- **行内式链接**：`[链接文字](URL "可选的标题")`
```markdown
[百度](https://www.baidu.com)
```
- **链接加标题**（鼠标悬停时显示）：
```markdown
[百度](https://www.baidu.com "点击前往百度")
```

---

### 6. 图片
语法和链接非常像，只是在前面多加一个感叹号 `!`。
```markdown
![替代文字](图片URL "可选的标题")
```

---

### 7. 引用
在段落前面加上 `>` 符号。
```markdown
> 这是一条引用的内容。
> 可以跨多行。
>>
>> 还可以嵌套引用。
```

---

### 8. 代码

- **行内代码**：用反引号 `` ` `` 包裹。
```markdown
使用 `print("Hello")` 函数输出。
```

- **代码块**：用三个反引号 ` ``` ` 包裹，并可以在开头指定编程语言（实现语法高亮）。
````markdown
```python
def hello():
    print("Hello, World!")
```
````

---

### 9. 分割线
使用三个或以上的 `-`、`*` 或 `_` 单独成行。
```markdown
---
***
___
```

---

### 10. 任务列表（待办清单）
常用于 GitHub 或笔记软件。
```markdown
- [x] 已完成的任务
- [ ] 未完成的任务
```

---

### 11. 转义字符
如果你想显示原本有特殊含义的符号（如 `*`、`#` 等），可以在前面加反斜杠 `\`。
```markdown
\* 这不是斜体，而是星号
\# 这不是标题
```

---

### 12. 字体颜色
最主流且兼容性最好的方法是使用 `<span>` 标签。你可以像下面这样指定颜色：

```html
<span style="color: 颜色值;">这里是你要变色的文字</span>
```

其中的“颜色值”有几种常用写法：

| 颜色值类型         | 语法示例                                               | 说明                                          |
| ------------- | -------------------------------------------------- | ------------------------------------------- |
| **颜色英文名**     | `<span style="color: red;">红色文字</span>`            | 简单直观，比如 `red`, `blue`, `green`, `orange` 等。 |
| **十六进制代码**    | `<span style="color: #FF0000;">红色文字</span>`        | 最精确，能表示千万种颜色。格式为 `#` 加上6位十六进制数（RRGGBB）。     |
| **RGB/RGBA值** | `<span style="color: rgb(255, 0, 0);">红色文字</span>` | 通过红绿蓝三原色的数值(0-255)来控制颜色，RGBA还可以额外指定透明度。     |

---

### 13. 下划线
 使用 `<u>` 标签（最推荐）

```markdown
<u>这是带下划线的文字</u>
```

显示效果：<u>这是带下划线的文字</u>

---

### 14. 引用块   #进阶学习

这是 **Obsidian Callout（标注/引用块）** 语法，是 Obsidian 对 Markdown 的扩展功能，不是标准 Markdown。

#### 基本语法

```markdown
> [!question] 可选标题
> 内容写在这里
```

**效果预览：**

> [!question] 可选标题
> 内容写在这里

#### 支持的 Callout 类型（11种）

| 类型 | 用途 | 渲染颜色 |
|------|------|---------|
| `[!note]` | 普通备注 | 蓝 |
| `[!info]` | 信息提示 | 蓝 |
| `[!tip]` / `[!hint]` / `[!important]` | 小贴士 | 绿 |
| `[!warning]` / `[!caution]` / `[!attention]` | 警告提醒 | 橙/红 |
| `[!danger]` | 危险警告 | 红 |
| `[!question]` / `[!help]` / `[!faq]` | 疑问/问答 | 绿 |
| `[!success]` / `[!done]` / `[!check]` | 已完成 | 绿 |
| `[!failure]` / `[!fail]` / `[!missing]` | 失败 | 红 |
| `[!example]` | 示例 | 紫 |
| `[!quote]` / `[!cite]` | 引用 | 灰 |
| `[!bug]` | Bug 记录 | 红 |

#### 高级用法

**可折叠 Callout：**
```markdown
> [!question]- 点我展开
> 折叠内容
```

> [!question]- 点我展开
> 折叠内容

**嵌套 Callout：**
```markdown
> [!question] 一级问题
> 内容
>
> > [!note] 补充说明
> > 嵌套细节
```

> [!question] 一级问题
> 内容
>
> > [!note] 补充说明
> > 嵌套细节

### 15. 双链（Wiki链接 / 内部链接） #进阶学习

双链是 **Obsidian 的核心功能**，也是构建"第二大脑"知识网络的基础。它让你在笔记之间建立双向链接，形成知识图谱。

---

#### 基本语法：`[[笔记名]]`

用双方括号包裹笔记名（**不需要写 `.md` 后缀**）：

```markdown
[[Fund Financial]]
[[牛奶可乐经济学]]
```

> 如果笔记名有空格，直接写空格即可：`[[投资笔记 2025]]`

---

#### 显示别名：`[[笔记名|显示文字]]`

如果你不想显示完整的笔记名，可以用 `|` 自定义显示文字：

```markdown
[[Fund Financial|基金金融笔记]]
[[纳瓦尔宝典|Naval]]
```

显示为：[基金金融笔记]()

---

#### 链接到标题：`[[笔记名#标题]]`

可以链接到另一篇笔记中的**具体章节**：

```markdown
[[Fund Financial#夏普比率]]
[[20250630#1. 单位净值]]
```

> 当渲染时，鼠标悬停会**预览**该标题下的内容。

---

#### 链接到块：`[[笔记名#^块ID]]`

精确链接到某个**段落**（行尾添加 `^块ID` 作为锚点）：

```markdown
在目标段落末尾添加锚点：
这是一段重要的定义。^my-def

在其他笔记中引用：
[[20250630#^my-def]]
```

> 块 ID 可以手动指定（如 `^my-def`），也可以让 Obsidian 自动生成（输入 `[[^^` 后选择段落）。

---

#### 嵌入内容：`![[笔记名]]`

在 `[[` 前加一个感叹号 `!`，就能把另一篇笔记的**全部内容**嵌入到当前笔记中：

```markdown
![[20250630]]
```

**嵌入部分内容：**

```markdown
![[20250630#1. 单位净值]]
![[20250630#^my-def]]
```

> 嵌入的内容在**阅读模式**下实时渲染，修改源笔记后嵌入处会自动更新。

---

#### 嵌入图片：`![[图片名.png]]`

图片也可以嵌入显示（而非显示链接文本）：

```markdown
![[最大回撤.png]]
```

> Obsidian 支持 PNG、JPG、GIF、WebP 等常见格式。

---

#### 路径写法：`[[文件夹/笔记名]]`

如果笔记在子文件夹中，可以写相对路径：

```markdown
[[June/20250630]]
[[attachment/最大回撤.png]]
```

---

#### 创建新笔记：`[[不存在的笔记]]`

当你输入一个**尚未创建**的笔记名时：

```markdown
[[7月投资复盘]]
```

- Obsidian 会以**灰色/淡化**显示（表示链接断裂）
- 点击该链接 → 自动创建新笔记
- 这是一种"先链接、后创建"的工作流

---

#### 查看反向链接

双链不只是"从 A 到 B"，也是"从 B 到 A"：

- 在笔记底部，Obsidian 会自动显示**反向链接面板**（Backlinks）
- 显示**有哪些笔记链接到了当前笔记**
- 还会显示**未链接的提及**（写了笔记名但没有用 `[[ ]]` 包裹的地方）

---

#### 知识图谱（Graph View）

当你大量使用双链后，点击左侧的 **Graph View（图谱视图）** 按钮：

- 每个笔记是一个**节点**
- 每条链接是一条**线**
- 节点越大 = 被引用的次数越多（中心度越高）
- 可以筛选标签、搜索特定笔记

---

#### 完整示例

```markdown
# 今日学习笔记

## 基金知识
- 复习了 [[Fund Financial#夏普比率|夏普比率]] 的计算方法
- 对比了 [[Fund Financial#卡玛比率]] 的区别
- 参考了 ![[20250630#1. 单位净值]] 中的定义

## 读书笔记
- [[牛奶可乐经济学#机会成本]] 这一章很有启发
- 相关图片：![[最大回撤.png]]
```

---

#### 总结

| 语法 | 作用 | 示例 |
|------|------|------|
| `[[笔记名]]` | 基本链接 | `[[Fund Financial]]` |
| `[[笔记名\|别名]]` | 自定义显示文字 | `[[Fund Financial\|金融笔记]]` |
| `[[笔记名#标题]]` | 链接到章节 | `[[Fund Financial#夏普比率]]` |
| `[[笔记名#^块ID]]` | 链接到段落 | `[[20250630#^定义]]` |
| `![[笔记名]]` | 嵌入整篇笔记 | `![[20250630]]` |
| `![[图片.png]]` | 嵌入图片 | `![[最大回撤.png]]` |
| `[[文件夹/笔记名]]` | 跨文件夹链接 | `[[June/20250630]]` |

### 16. 笔记属性（Properties / YAML Frontmatter） #进阶学习

笔记属性是 Obsidian 的**元数据系统**，让你为每篇笔记添加结构化信息（如日期、标签、分类、评分等）。本质是在笔记开头用 YAML 格式定义"键值对"。

#### 基本语法：三个减号包裹

在笔记的**最开头**（第 1 行），用两组 `---` 包裹属性内容：

```markdown
---
title: 我的笔记标题
date: 2025-07-01
tags:
  - 投资
  - 学习
status: 进行中
---

正文从这里开始……
```

> ⚠️ **必须是笔记的第一行**，前面不能有空行或其他内容。

---

#### 支持的属性类型

| 类型 | 写法示例 | 说明 |
|------|---------|------|
| **文本** | `title: 我的笔记` | 纯文本，不需要引号（含特殊字符时需引号） |
| **数字** | `rating: 8` | 整数或小数 |
| **日期** | `date: 2025-07-01` | YYYY-MM-DD 格式 |
| **时间** | `time: 14:30` | HH:MM 格式 |
| **复选框** | `done: true` | `true` 或 `false` |
| **行内列表** | `tags: [投资, 学习]` | 方括号包裹，逗号分隔 |
| **多行列表** | 见下方 | 用 `-` 逐行列出的方式 |

---

#### 列表的两种写法

**行内写法（紧凑）：**
```yaml
tags: [obsidian, markdown, 教程]
```

**多行写法（清晰）：**
```yaml
tags:
  - obsidian
  - markdown
  - 教程
```

> 两种写法效果相同，多行写法可读性更好。

---

#### 嵌套属性

属性值也可以是对象（嵌套结构），用于更复杂的元数据：

```yaml
---
book:
  title: 纳瓦尔宝典
  author: Naval Ravikant
  rating: 9
  read_date: 2025-06-15
---
```

---

#### 与 Dataview 结合（核心用法 ⭐）

属性 + Dataview 是 Obsidian 的强大组合。Dataview 可以根据属性值**查询和汇总**笔记。

**示例：汇总所有"进行中"的学习笔记**

````markdown
```dataview
TABLE date, tags
FROM "学习笔记"
WHERE status = "进行中"
SORT date DESC
```
````

**示例：统计书籍评分**

````markdown
```dataview
TABLE book.rating AS "评分", book.author AS "作者"
FROM "读书笔记"
SORT book.rating DESC
```
````

---

#### 常用属性命名建议

| 用途 | 推荐属性名 | 示例值 |
|------|-----------|-------|
| 日期 | `date`, `created`, `updated` | `2025-07-01` |
| 分类 | `category`, `type` | `读书笔记` |
| 标签 | `tags` | `[投资, 技术]` |
| 状态 | `status` | `草稿 / 进行中 / 已完成` |
| 评分 | `rating` | `8` |
| 来源 | `source` | `微信公众号 / 书籍 / 课程` |
| 别名 | `aliases` | `[moc, 地图]` |

---

#### 完整示例

```markdown
---
title: 基金投资入门
date: 2025-07-01
tags:
  - 投资
  - 基金
  - 入门
status: 已完成
rating: 8
source: 某理财课程第3讲
aliases:
  - Fund Basics
  - 基金基础
---

# 基金投资入门

正文内容……
```

> 💡 `aliases` 是 Obsidian 的**内置属性**，定义笔记别名后，可以用 `[[别名]]` 引用这篇笔记。

---

#### 总结

属性让你从"乱糟糟的文件夹"升级到**可查询的知识库**。配合 Dataview，你可以像操作数据库一样管理自己的笔记。
